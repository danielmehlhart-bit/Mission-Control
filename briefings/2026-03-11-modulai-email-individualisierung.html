Einladungs-E-Mails individualisieren

Aktuell verschickt das System die Standard-Auth-E-Mails von Supabase — generisch, ungebrandeted. So ändern wir das.

Problem: Auth-Mails kommen von einer @supabase.io-Adresse, ohne Branding, ohne Ton der App. Das wirkt unprofessionell — besonders wenn du Mandanten damit onboardest.
🌐
Option A — Eigene E-Mail-Domain einrichten
Erster Schritt · Empfohlen

E-Mails kommen dann von z.B. no-reply@mehlhart.de statt einer generischen Supabase-Adresse. Verbessert Zustellbarkeit und Vertrauen erheblich.

  • Sender-Domain einrichten (z.B. notify.mehlhart.de)
  • 3 DNS-Einträge setzen (SPF, DKIM, Return-Path)
  • Danach kommen alle Auth-Mails von deiner Domain
Voraussetzung: Eigene Domain + Zugang zu den DNS-Einstellungen
🎨
Option B — Gebrandete E-Mail-Templates
Aufbauend auf Option A
★ Empfohlen

Nach Domain-Setup können wir den kompletten E-Mail-Inhalt auf ModulAI anpassen — kein generisches Supabase-Layout mehr.

  • Violet-Branding passend zum Design-System (Primary #7C3AED)
  • Inter-Font, dunkle Schrift, weiße Karten
  • Deutsch, Du-Form, gleicher Ton wie die App
  • Büroname im Header statt „Lovable"
  • Logo einbinden falls vorhanden
  • Gilt für: Einladung, Passwort-Reset, E-Mail-Bestätigung, Magic Link
🔧
Option C — Einladungs-Flow im Frontend anpassen
Ohne eigene Domain · Fallback

Wenn keine eigene Domain gewünscht — zumindest das Frontend-Erlebnis verbessern.

  • Passwort-Setzen-Screen (/passwort-setzen) im ModulAI-Branding
  • Einladungs-Texte im Frontend verbessern (Willkommens-Nachricht nach erstem Login)
  • Onboarding-Flow nach der Einladung einbauen

🏔️ Hattis Empfehlung

Option A + B zusammen — eigene Domain + gebrandete Templates. Professionellster Weg, stärkster Eindruck bei Mandanten-Onboarding. DNS-Setup ist einmalig, danach läuft alles automatisch.

⏱ ca. 10 min Setup + DNS-Propagation (1–24h)
