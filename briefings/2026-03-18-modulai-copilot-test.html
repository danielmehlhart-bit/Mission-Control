Root Cause Analyse
🔴 Kernproblem: Edge Function liefert nur Aggregat-Daten
Die Edge Function project-copilot gibt der AI nur zusammengefasste Projekt-Daten (Status, Task-Count) — keine einzelnen Tasks, keine Fälligkeitsdaten, keine Assignees. Das erklärt:
• Warum Responses generisch sind und keine Task-Details kennen
• Warum keine Links entstehen (kein Task-Objekt → keine task_id → kein Link möglich)
• Warum Queries zu Überfälligkeit oder Verantwortlichen scheitern
ℹ️ Zum Navigation-Bug (Daniel, manuell beobachtet)
Der Bug "Klick auf Task → landet auf Aufgaben-Board" kommt nicht aus Links in der Prose-Response (keine vorhanden), sondern vermutlich aus den suggested_actions Buttons die unter der Antwort erscheinen. Diese Actions (z.B. "Aufgabe anzeigen") navigieren zur /aufgaben-Route ohne Copilot-Kontext zu erhalten. Das ist ein echter UX-Bug der behoben werden sollte: suggested_actions sollten Tasks in einem Drawer/Modal öffnen, nicht per Page-Navigation.
Empfohlene Fixes
Edge Function: Einzelne Tasks übergeben 🔴 Hoch Tasks mit id, title, due_date, assignee, status in den AI-Kontext einbauen (statt nur counts). Dann kann AI auch Fälligkeiten und Verantwortliche nennen.
Task-IDs in Response → klickbare Chips 🔴 Hoch CopilotMessage-Komponente: Task-IDs aus AI-Response parsen → als klickbare Badge/Chip rendern → öffnet Task-Drawer INNERHALB Copilot (kein Page-Wechsel).
Suggested Actions: Drawer statt Navigation 🔴 Hoch Alle suggested_actions die Tasks öffnen: router.push('/aufgaben') ersetzen durch openTaskDrawer(task_id). User bleibt im Copilot-Kontext.
