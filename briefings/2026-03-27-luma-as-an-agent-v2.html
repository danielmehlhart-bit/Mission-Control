Section 1

IST vs. SOLL: Wo wir stehen und wo wir hinwollen

Heute · Der Assistent

Was wir haben ✅

  • Solide Basis: Sprach-zu-Aktion für Tasks & Bautagebuch
  • Co-Pilot Chat über statische project-knowledge.md
  • Beantwortet Fragen: "Was ist ein Bautagebuch?"
⚠️ Limitierung: Er kann nur Fragen beantworten, keine Befehle ausführen. "Finde Aufgabe X und weise sie Y zu" — geht nicht. Er ist eine Frage-Antwort-Maschine, kein Macher.
Morgen · Der Agent

Was wir wollen 🎯

  • Der Chat wird zum zentralen Hub der App
  • Komplexe, mehrstufige Befehle in natürlicher Sprache
  • Agent versteht, plant und führt Aktionen aus
  • LUMA-Skills werden kombiniert — dynamisch
🧠
Section 2

Die Brücke: Was fehlt zum echten Agenten?

Das Schlüsselkonzept: Reason + Act (ReAct)

Der aktuelle Co-Pilot kann denken, aber nicht handeln. Ein echter Agent kann beides — in einer Schleife.

Befehl empfangen
→
🤔 Denken
→
🔧 Werkzeug aufrufen
→
📊 Ergebnis auswerten
→
🤔 Wieder denken
→
✅ Ziel erreicht

Der ReAct-Loop läuft so lange, bis das Ziel erfüllt ist — vollständig autonom.

🧰

Werkzeugkiste (Skills)

Jede Funktion in LUMA muss als eigenständiges, aufrufbares Werkzeug existieren — sauber gekapselt, einzeln testbar.

🧠

Gehirn (Agenten-Laufzeit)

Eine Logik-Schicht, die einen Befehl in Schritte zerlegt und entscheidet, welches Werkzeug wann und wie benutzt wird.

🗺️
Section 3

Die angepasste Roadmap: Der Weg zum Co-Piloten

1
Phase 1 · Fundament

🧰 Werkzeuge schmieden — API-fizierung aller Funktionen

  • Jede Aktion in LUMA wird in eine saubere Supabase Edge Function gekapselt
  • Beispiele: create_task, edit_project, find_user, add_comment, upload_file
  • Jede Funktion hat klaren Input, Output und Fehlerbehandlung
🎯 Ziel: Eine vollständige "Werkzeugkiste" an Skills — auf die ein Agent zugreifen kann. Bonus: Die Codebasis wird sauberer und besser testbar.
2
Phase 2 · Das Gehirn

🧠 Das Gehirn aufrüsten — Agenten-Laufzeit implementieren

  • Neue Supabase Edge Function /agent-v2 implementiert den ReAct-Loop
  • Frameworks wie LangChain oder LlamaIndex können helfen
  • Start mit 2-3 einfachen Skills: find_project + create_task
  • Der Loop wird iterativ getestet und validiert
🎯 Ziel: Der Agent löst erstmals eine komplexe Anfrage wie "Erstelle Aufgabe X im Projekt Y" — selbstständig, durch Kombination zweier Werkzeuge.
3
Phase 3 · Der Austausch

🔄 Integration & Rollout — Der Co-Pilot lebt

  • Frontend-Backend des Co-Pilot-Chats wird ausgetauscht
  • Statt alter Q&A-Funktion → neues /agent-v2 Backend
  • Skills aus Phase 1 werden schrittweise für den Agenten freigeschaltet
🎯 Ziel: Der "dumme" Assistent ist vollständig durch den "schlauen" Agenten ersetzt. Nutzer können komplexe, mehrstufige Aufgaben per Chat erledigen.

Section 4

Der entscheidende Unterschied

Assistent · Heute

Die Frage-Antwort-Maschine

→ User stellt eine Frage
→ LLM schaut in Wissens-DB
→ LLM antwortet mit Text
→ Ende. Keine Aktion.
Agent · Morgen

Der echte Co-Pilot

→ User gibt einen Befehl
→ Agenten-Gehirn denkt
→ Ruft Skill 1 auf → Daten
→ Denkt wieder
→ Ruft Skill 2 auf → Aktion
→ ✅ Agent meldet Erfolg
