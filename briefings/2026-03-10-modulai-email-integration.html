Lösung für Nicht-Gmail/M365: Inbound-Adresse

User (Strato / IONOS / Outlook) │ │ Weiterleitungsregel oder BCC ▼ inbox-<tenant>@modulai.app │ ▼ Inbound-Mail-Service (z.B. Postmark) │ Webhook (POST mit parsed Mail) ▼ Edge Function "process-inbound-email" │ ├── Projekt-Matching (Absender-Map) ├── AI-Summary + Task-Extraktion └── Speichern in email_inbox
Vorteile:
  • Null Support-Aufwand — User richtet selbst Weiterleitung ein
  • Funktioniert mit jedem Mail-Provider (Strato, IONOS, Outlook lokal, …)
  • Kein IMAP, keine TCP-Verbindungen, keine Credential-Speicherung
  • Self-Service: User bekommt Adresse angezeigt + Anleitung

Was zu tun ist (später)

  1. Domain für Inbound-Mail einrichten — z.B. modulai.app MX-Records
  2. Postmark/SendGrid Inbound-Stream konfigurieren — ~15 $/10k Mails
  3. Edge Function process-inbound-email bauen — Webhook-Empfänger
  4. Pro Tenant eine unique Adresse generieren + in UI anzeigen
  5. Anleitungs-Wizard für User — Weiterleitung bei Strato / IONOS / Outlook einrichten

Datenmodell

Für Inbound-Mails wird ein neuer Connection-Typ provider = 'inbound' ergänzt.
