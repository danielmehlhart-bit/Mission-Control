  • 😰
    Kognitive Überlastung: Eine leere App + eine lange To-Do-Liste = Schock. Architekten sind pragmatisch — sie wollen sofort loslegen, nicht erst einen Lehrplan abarbeiten.
  • 👻
    Kein "Aha-Moment": Checklisten zeigen TASKS, keine Features. Der User versteht nicht, warum ein Feature ihm das Leben einfacher macht. Besonders kritisch für Voice-Features, die man sehen (hören) muss.

    Zu generisch: "Erstelle dein erstes Projekt" klingt wie jedes andere PM-Tool. Kein spezifischer Wert für Architekten auf der Baustelle kommuniziert.
  • 📍
    Kein Kontext: Checklisten hängen irgendwo in der Sidebar. Guided Tours und Modals erscheinen dort, wo der User gerade ist — im richtigen Moment.

Abschnitt B

Die 3 wirkungsvollsten Onboarding-Patterns

Pattern 01

🖼️ Feature Spotlight Modal (Welcome Karussell)

Ein fullscreen- oder centered Modal beim allerersten Login — bevor der User die App überhaupt sieht. 3–6 Slides die die wichtigsten Features visuell zeigen: großes Icon, kurzer Titel, ein Satz Erklärung, Skip-Option. Kein Feature-Dumping — jede Slide hat einen klaren Punkt.

Wann es wirkt: Erster Login, neues Feature-Release, nach Re-Aktivierung
Vorbild: 🔷 Linear 🟣 Notion 🟠 Loom
💡 Für ModulAI: 5 Slides — Dashboard Overview → Projekte → 🎤 Voice Features (animiertes Mikro) → Tasks → Team Invite
Pattern 02

🗺️ Guided Product Tour (Schritt-für-Schritt)

Overlay-basierter Highlight: Ein Element der UI wird "spotlighted" (Rest abgedunkelt), daneben erscheint ein Tooltip mit Erklärung und "Weiter"-Button. Der User führt echte Aktionen durch — nicht nur lesen, sondern klicken. Aktivierungsrate 40–60% höher als passive Erklärungen.

Wann es wirkt: Nach dem Welcome Modal, on-demand über "Hilfe"-Button, bei neuen Features
Vorbild: 🔴 Figma 🟡 Asana 🔵 Intercom
💡 Für ModulAI: "Voice Tour" — 4 Steps: Mikrofon finden → Aufnahme starten → Memo wird zu Task → Task editieren
Pattern 03

🌱 Empty State Coaching

Leere Listen- und Dashboard-Bereiche sind die beste Werbefläche, die du hast. Statt "Noch keine Projekte" zeigen: ein aussagekräftiges Illustration + konkreter CTA ("Erstes Projekt anlegen") + kleiner Hinweis auf das Voice-Feature ("Oder einfach eintippen lassen — per Sprache 🎤"). Kein extra-Tool nötig, direkt in den Komponenten umsetzbar.

Wann es wirkt: Immer — besonders für neue User ohne Daten
Vorbild: ⬛ Linear 🟣 Notion 🔵 Slack
💡 Für ModulAI: Jede leere Liste hat ein Bau-/Architekten-Motiv + direkten CTA + 1 Satz Voice-Hinweis

Abschnitt D

Hattis Empfehlung für ModulAI

Solo-Founder-Perspektive: Kein Over-Engineering. Maximaler Impact, minimaler Aufwand. Daniel hat Lovable + Supabase + React — das ist ein verdammt guter Stack für alles was hier gebraucht wird.

1
Phase 1 — Diese Woche · 0 € · ~4–6 Stunden

Eigenes Welcome Modal bauen

Direkt in Lovable prompen: "Erstelle ein WelcomeModal.tsx mit 5 Slides und Supabase-Integration." Das ist kein Hexenwerk — React State + einfaches Modal + Supabase-Metadata-Update. Kein Drittanbieter, kein Abo, volle Kontrolle.

  • WelcomeModal.tsx mit 5 Slides bauen (Dashboard → Projekte → 🎤 Voice → Tasks → Team)
  • Trigger: user.created_at < 24h AND hasSeenOnboarding = false
  • Slide 3: Animiertes Mikrofon, Voice-Feature erklären — das ist das Killer-Feature
  • Beim Close: supabase.auth.updateUser({ data: { hasSeenOnboarding: true } })
  • Empty States in Projekt- und Task-Listen mit CTAs + Voice-Hinweis updaten
2
Phase 2 — Wenn >50 aktive User · 0 € · ~1 Wochenende

Driver.js oder React Joyride für Guided Tours

Wenn das Modal die User aktiviert hat, aber manche trotzdem nicht verstehen wie Voice funktioniert: Eine geführte 4-Step Tour einbauen. Auslöser: "Hilfe"-Button oder "Voice Tour starten" Link.

  • npm install driver.js — 20KB, kein React-Wrapper nötig
  • 4-Step Tour: Mikrofon-Button finden → Record → Auto-Task → Editieren
  • Analytics: welche Steps werden abgebrochen? → Weak Points finden
  • On-Demand: Tour jederzeit neustart-bar über Hilfe-Menü
3
Phase 3 — Wenn Churn ein Problem wird · ab $50/mo

SaaS-Tool evaluieren (UserGuiding / Inline Manual)

Erst wenn: Churn messbar steigt, A/B-Testing von Onboarding-Flows gebraucht wird, oder kein Dev-Bandwidth mehr für Onboarding-Maintenance. Dann Inline Manual ($50/mo) oder UserGuiding ($174/mo) testen — beide ohne Re-Deploy aktualisierbar.

  • Trigger: Churn > 15% monatlich ODER >200 MAUs
  • Inline Manual first — günstigster Einstieg, auch Help Center dabei
  • Appcues / Pendo: Skip. Zu teuer, zu komplex für diese Phase.

Abschnitt F

Voice Feature Spotlight — Das Killer-Feature

🎤

Sprach-Memo aufnehmen

Einfach auf das Mikrofon tippen — ModulAI macht daraus eine strukturierte Aufgabe. Hände frei, Gedanken gesichert.

🏗️ Architekten sind auf der Baustelle
🧤 Hände oft schmutzig / besetzt
⚡ 10× schneller als Tippen
🧠 Gedanken sofort festhalten
🤖 AI wandelt Sprache in Tasks um

⚠️ Warum das Onboarding HIER besonders wichtig ist

Voice-Features sind unsichtbar, solange man sie nicht kennt. Ein Benutzer der noch nie aufs Mikrofon getippt hat weiß nicht, dass es da ist. Das Welcome Modal muss dieses Feature aktiv demonstrieren — nicht nur erwähnen. Slide 3 sollte deshalb das animierte Mikrofon (pulsierende Welle) + einen Satz haben: "Tippe einfach auf das Mikrofon-Icon — ModulAI erkennt deine Sprache und erstellt automatisch einen Task."

💡 Konkrete Slide 3 Gestaltung

  • Großes animiertes Mikrofon-Icon mit Pulsiereffekt (CSS animation, inline umsetzbar)
  • Titel: "Sprach-Memo — dein Baustellen-Helfer"
  • Body: "Hände voll? Einfach auf 🎤 tippen, kurz sprechen — ModulAI erstellt automatisch eine Aufgabe."
  • Optional: Kleines "Try it"-Badge → direkt zur Voice-Funktion springen
  • Hintergrundfarbe dieser Slide: leichtes Blau (#EEF3FF) zur Differenzierung
