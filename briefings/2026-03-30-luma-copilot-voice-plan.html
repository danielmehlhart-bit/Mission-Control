Copilot Voice Integration & Agent Use Cases

Was schon geht, was fehlt, wie wir Voice direkt in den Copilot-Chat verdrahten — und welche Use Cases als nächste kommen.

🎙️ Voice → Copilot 🤖 Agent Use Cases ⚡ 1-Line Fix 📅 30. März 2026

🔍 Status Quo — Was wir gesehen haben

✓ Bereits vorhanden

  • Mic-Button in CopilotChat existiert
  • Transcription via transcribe-audio Edge Function läuft
  • Text wird nach Aufnahme korrekt empfangen
  • Floating Voice FAB für Task/Bautagebuch funktioniert
  • Copilot beantwortet komplexe Queries live (Screenshots bestätigt)
  • Suggested Actions werden generiert

✗ Das Gap

  • Nach Transkription: Text landet nur im Input-Feld
  • Nutzer muss manuell auf Senden drücken → Bruch im Flow
  • Floating FAB auf Copilot Mobile bewusst versteckt
  • Kein visuelles Feedback während Aufnahme im Chat
  • Voice → Copilot nicht end-to-end verbunden

⚡ Die Fixes

1

Auto-Submit nach Transkription

1-Line Fix

In CopilotChat.tsx — Funktion transcribeAudio: Statt den Text ins Input-Feld zu legen, direkt onSendMessage aufrufen. Der Nutzer spricht → Text wird transkribiert → Copilot antwortet. Kein manueller Schritt.

src/components/copilot/CopilotChat.tsx → transcribeAudio()
// vorher: Text ins Input-Feld legen, Nutzer muss selbst senden if (data.text) { setInput(data.text); } // nachher: direkt senden + kurz im Input anzeigen (Review-Moment) if (data.text) { setInput(data.text); // kurz anzeigen onSendMessage(data.text); // sofort abschicken setTimeout(() => setInput(""), 300); }

Optional: 800ms Delay vor Auto-Submit, damit der Nutzer kurz sieht was transkribiert wurde und noch abbrechen kann.

2

Recording-State im Chat sichtbar machen

UX

Während der Nutzer spricht, soll der Mic-Button pulsieren und ein Hinweis „Aufnahme läuft…" erscheinen — ähnlich wie beim Voice FAB. Gibt Sicherheit dass die Aufnahme wirklich aktiv ist.

// Button-State während Aufnahme <Button className={cn( "h-10 w-10 p-0 shrink-0", isRecording && "bg-red-100 text-red-600 animate-pulse" )} onClick={isRecording ? stopRecording : startRecording} > {isRecording ? <Circle className="h-4 w-4" fill="currentColor" /> : <Mic className="h-4 w-4" />} </Button>
3

Floating FAB auf Copilot-Seite enablen (Mobile)

Optional

Aktuell ist der große Voice FAB auf der Copilot-Seite auf Mobile versteckt. Das macht Sinn — dort soll Voice direkt in den Chat gehen, nicht einen Task-Dialog öffnen. Fix 1 + 2 lösen das eleganter als den FAB zu enablen.

→ Empfehlung: Fix 1 + 2 reichen. FAB auf Copilot bleibt versteckt.
