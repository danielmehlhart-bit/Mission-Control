01 — Executive Summary

Die App ist funktional und das visuelle Design ist solide — saubere Typografie, konsistente Brand-Farbe. Es gibt jedoch signifikante UX-Probleme, besonders auf Mobile, die vor einem Kunden-Rollout gefixt werden müssen.

15 Screenshots wurden analysiert (Desktop 1440×900 + Mobile iPhone 14, 390×844 Viewport). Getestet wurden alle zentralen User Flows.

🔐 Login 📊 Dashboard 📋 Aufgaben-Board (Kanban) ➕ Task-Erstellung 📄 Task-Detail 📁 Dokumente 🤖 Copilot 📱 Bottom-Navigation
02 — Desktop Findings
2.1 Dashboard
Startseite nach Login
✅ Gut
  • ✅Übersichtliches Card-Grid mit 8 Projekten
  • ✅Klare Greeting-Personalisierung
  • ⚠️Kein Suchfeld, keine Filter-Optionen bei wachsender Projektanzahl — kritischer Gap ab ~12+ Projekten
  • ⚠️"+8 weitere →" Links zu klein als Klickziel — Mindest-Klickziel 44×44pt nicht erfüllt
2.2 Aufgaben-Board / Kanban
Kern-Workflow: Projektaufgaben verwalten
🟡 Probleme
  • ✅4-Spalten Kanban funktioniert grundsätzlich
  • ✅"25 weitere laden" Pagination vorhanden
  • 🔴"Offen" hat 45 Items → massive Spalten-Imbalance, erfordert endloses Scrollen
  • 🟡Status-Dropdown auf jeder Card UND Spalten-Zuordnung = redundantes Dual-Control. Untergräbt die Kanban-Metapher.
  • 🟡Keine Drag-and-Drop-Affordance sichtbar — User weiß nicht, dass Cards draggable sind
  • 🟡Keine Sortierung innerhalb der Spalten (nach Datum, Priorität, Projekt)
  • 🔴Kein Suchfeld auf der Aufgaben-Seite — bei 45+ offenen Tasks inakzeptabel
2.3 Neue Aufgabe Dialog
Task-Erstellungs-Formular
🟠 Probleme
  • ✅Klare Button-Hierarchie (Abbrechen vs. Erstellen)
  • 🟠Kein Zuweisungs-Feld (Assignee) im Erstellungs-Dialog — muss nachträglich editiert werden
  • 🟠Kein Phasen-Feld (Leistungsphase) — obwohl Phasen überall in der App angezeigt werden
  • 🟠Status-Feld zeigt "Offen" als disabled Input ohne Erklärung — sieht kaputt aus
  • 🟠Validierung nur per Toast unten rechts — keine Inline-Fehler am Feld (roter Rand + Fehlertext fehlt)
  • 🟠3 verschiedene Datumsformate in der App: "dd/mm/yyyy" (Dialog) · "2026-04-20" (Cards) · "20.03." (Dashboard)
2.4 Task-Detail Dialog
Aufgaben-Detailansicht
✅ Funktional
  • ✅Tabs für Details, Kommentare, Dateien vorhanden
  • ✅Titel editierbar
  • ✅Subtasks-Section mit Eingabefeld
  • ✅Speichern-Button sichtbar
  • 🟡Dialog-Inhalt muss gescrollt werden um Subtasks zu sehen — wichtige Section Below the Fold
03 — Mobile Findings (iPhone 14 · 390×844)
📱 Viewport: 390×844px  ·  hasTouch: true  ·  iPhone 14 Simulation
3.1 Bottom Navigation
Primäre Navigation auf Mobile
🔴 Kritisch
  • 🔴7 Items in der Tab-Bar (Übersicht, Aufgaben, Gedanken, Sprache/Mic, Copilot, Finanzen, Profil) — Apple HIG empfiehlt max 5
  • 🔴Touch-Targets ~40–45pt — unter dem Apple-Minimum von 44pt
  • 🔴"Profil" wird truncated/abgeschnitten auf 390px
  • 🟠Mikrofon-Button in der Mitte bricht das Tab-Bar-Muster — ist es Navigation oder Aktion?
  • 🟠"Dokumente" hat KEINEN Tab, obwohl für Bau-PM essentiell — dafür hat "Gedanken" einen Tab
3.2 Aufgaben-Liste Mobile
Task-Board auf kleinem Viewport
🟡 Probleme
  • 🟠Filter-Bereich nimmt ~40% des Viewports ein (Button + 2 Dropdowns + 4 Status-Chips) — nur ~1.5 Task-Cards sichtbar ohne Scrollen
  • 🟠Status-Filter-Chips ~30pt Höhe — unter dem 44pt Touch-Minimum
  • 🟡Prioritäts-Labels zeigen "Normal**" / "Hoch**" — Rendering-Bug mit Sternchen (Markdown nicht gerendert)
  • 🟡Inline-Status-Dropdown auf jeder Card ist auf Mobile fehleranfällig — versehentliche Taps beim Scrollen
3.3 Neue Aufgabe Mobile
Task-Erstellung als Drawer/Sheet
🟡 Probleme
  • ✅Drawer/Sheet Pattern wird korrekt verwendet
  • 🟠X-Button zum Schließen nur ~24px — drastisch unter Mindest-Touch-Target
  • 🟡Textarea hat Desktop-Resize-Handle auf Mobile — sinnlos und irritierend
  • 🟡Validierungs-Toast am unteren Bildschirmrand — kollidiert potenziell mit iPhone Home Indicator (Safe Area)
  • 🟡Keine Inline-Validierung am Feld
3.4 Dokumente
Datei-Verwaltung
🟠 Dead End
  • 🟠Leere Seite: "Dokumente werden von Teammitgliedern über die Projektseite hochgeladen" — ohne Button, ohne Link
  • 🟠Kein Upload-CTA, kein Link zur Projektseite — Nutzer ist gestrandet ohne Handlungsoption
  • 🟠Empty State ist eine Sackgasse (Dead End) — klassisches UX-Anti-Pattern
3.5 Copilot
KI-Feature
🔴 Kritisch
  • 🔴Seite ist KOMPLETT LEER — kein Input-Feld, keine Prompts, kein Onboarding, kein Placeholder
  • 🔴Bottom-Navigation fehlt auf dieser Seite — Nutzer ist ohne Rückweg (nur OS-Geste)
  • 🔴Für ein AI-Feature als zentrales Differenzierungsmerkmal ist das inakzeptabel
04 — Übergreifende Probleme
4.1 Datumsformate (3 verschiedene!)
  • 📍Dashboard: "20.03." — deutsch, ohne Jahr
  • 📍Kanban-Cards: "2026-04-20" — ISO 8601
  • 📍Dialog: "dd/mm/yyyy" — Browser-Default
→ Empfehlung: dd.mm.yyyy (deutsch) überall vereinheitlichen
4.2 Onboarding-Modal
  • 🟡Modal + Banner gleichzeitig = doppeltes Onboarding für dasselbe Feature
  • 🟡3 Dismiss-Optionen: X, "Nicht mehr anzeigen", "Später" — Decision Fatigue
  • 🟡Modal blockiert sofort nach Login — kein Orientieren möglich
  • 🟡localStorage-Bug: Dismiss wurde nicht persistent gespeichert — Modal erschien bei Tests wiederholt
4.3 Kein globales Suchfeld
  • 🔴Weder auf Dashboard noch auf Aufgaben-Seite gibt es ein Suchfeld
  • 🔴Bei 45+ offenen Tasks und 8+ Projekten ist das ein kritischer Gap
4.4 Console-Fehler
NotFoundError: Failed to execute
'removeChild' on 'Node':
The node to be removed is not
a child of this node.

Onboarding-Modal verursacht DOM-Konflikte wenn programmatisch entfernt.
